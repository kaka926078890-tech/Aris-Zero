# Aris-Zero（纯净版）

本仓库为 **Aris 的纯净发行版**：仅包含源码与配置，不包含对话/运行过程中生成的数据。你可以在本机安装后，通过简单配置即可使用 Aris 桌面 AI 伴侣。

---

# 零基础安装与使用指南

如果你**从未写过代码、没用过命令行**，按下面步骤一步步做即可。Aris 是一款桌面上的 AI 伙伴：有一个 3D 小窗口，你可以和它对话，它也会偶尔主动和你说话；支持记忆与个性设置。

---

## 第一步：准备所需软件（共三项，第三项可跳过）

### 1. 安装 Node.js（让电脑能运行 Aris）

- **Node.js** 是一个「运行环境」，可以理解为让 Aris 能在你电脑上跑起来的必要软件。
- **下载**：打开浏览器，访问 [https://nodejs.org](https://nodejs.org) ，选择页面上的 **LTS 版本**（推荐），点击下载。
- **安装**：下载完成后双击安装包，一路点「下一步」即可（默认选项不用改）。安装完成后**关闭并重新打开**你的终端/命令提示符（如果有开的话）。

**如何确认装好了？**  
打开「终端」（Mac）或「命令提示符 / PowerShell」（Windows），输入下面这一行并回车：

```text
node -v
```

如果出现类似 `v20.x.x` 或 `v18.x.x` 的版本号，说明已安装成功。

---

### 2. 获取 DeepSeek API 密钥（让 Aris 能「思考」和回复）

- Aris 的对话能力依赖 **DeepSeek** 的接口，需要你在 DeepSeek 官网申请一个 **API Key**（密钥）。
- **打开**：[https://platform.deepseek.com](https://platform.deepseek.com) （或搜索「DeepSeek 开放平台」）。
- **注册/登录** 后，在控制台里找到 **API Keys** 或「创建密钥」。
- **创建密钥** 并**复制**下来（形如 `sk-xxxxxxxx...`），保存到记事本里备用。  
  ⚠️ 密钥只显示一次，务必妥善保存；不要分享给他人。

---

### 3. 安装 Ollama（**可跳过**）

- **不装也能正常用 Aris**，只是「记忆」能力会弱一些（例如对你说过的话、偏好等记得没那么牢）。若你希望 Aris **更会记住**你的习惯和过往对话，建议安装。
- **下载**：打开 [https://ollama.com](https://ollama.com) ，下载并安装 Ollama（和装普通软件一样，一路下一步即可）。
- **安装完成后**，打开终端（或命令提示符），输入下面一行并回车，等待下载完成：
  ```bash
  ollama pull nomic-embed-text
  ```
- 若你**跳过**本步，直接进行第二步即可；以后想加强记忆再回来装也可以。

---

## 第二步：获取 Aris-Zero 并解压

- 打开本项目的 GitHub 页面，点击 **Code** → **Download ZIP**，下载压缩包。
- 将 ZIP 解压到你容易找到的文件夹，例如「桌面」或「文档」下的 `Aris-Zero` 文件夹。  
  记住这个文件夹的路径，例如：`C:\Users\你的用户名\Desktop\Aris-Zero` 或 `/Users/你的用户名/Desktop/Aris-Zero`。

---

## 第三步：打开「终端」并进入项目文件夹

- **Windows**：在 Aris-Zero 文件夹里，按住 `Shift` 键再**右键点击**空白处，选择「在此处打开 PowerShell 窗口」或「在终端中打开」。  
  或者：按 `Win + R`，输入 `cmd` 回车，然后输入 `cd` 加一个空格，再把 Aris-Zero 文件夹**拖进**窗口后回车。
- **Mac**：打开「终端」（在「启动台」或「应用程序 → 实用工具」里），输入 `cd` 加一个空格，把 Aris-Zero 文件夹**拖进**终端窗口，回车。

此时终端里显示的路径应包含 `Aris-Zero`，表示你已经在这个项目里了。

---

## 第四步：安装依赖（第一次必须做）

在终端里**逐行**输入下面两条命令，每输入一行按一次回车，等待执行完成再输入下一条：

```bash
npm install
```

这会自动下载 Aris 运行所需的组件，可能需要一两分钟，出现很多文字是正常的。  
完成后不要关闭终端。

---

## 第五步：配置 API 密钥

1. 在 Aris-Zero 文件夹里，找到名为 **`.env.example`** 的文件（若看不到，可能是被隐藏了，需在系统设置里显示「隐藏的文件」）。
2. **复制**这个文件，在同一文件夹里粘贴一份，并把**新复制的那份**改名为 **`.env`**（注意前面有个点，没有 .example）。
3. 用**记事本**（Windows）或**文本编辑**（Mac）打开 **`.env`**（不要用 Word）。
4. 找到这一行：
   ```text
   DEEPSEEK_API_KEY=sk-...
   ```
   把 `sk-...` 替换成你在第二步里复制的**完整密钥**，例如：
   ```text
   DEEPSEEK_API_KEY=sk-你复制的密钥
   ```
5. **保存**后关闭。同一文件里的 `DEEPSEEK_API_URL` 一般不用改，保持 `https://api.deepseek.com` 即可。

---

## 第六步：启动 Aris

在**同一个终端窗口**里（确保还在 Aris-Zero 文件夹下），输入：

```bash
npm start
```

回车后稍等几秒，会弹出一个**透明 3D 小窗口**，那就是 Aris 的主界面。

- **和 Aris 对话**：用鼠标**点击窗口正中间**，会弹出输入框，输入你想说的话，按**回车**发送。
- **关闭对话层**：再点一下空白处或按 Esc 即可收起。
- **关闭 Aris**：直接关掉该窗口，或从窗口左上角/菜单栏退出。

以后每次想用 Aris，都是：打开终端 → 进入 Aris-Zero 文件夹（用第三步的 `cd` 方式）→ 输入 `npm start` 回车。

---

## 常用操作一览（不用记，用的时候来查）

| 你想做的事           | 怎么操作 |
|----------------------|----------|
| 和 Aris 说话         | 点击 3D 窗口**中心**，在输入框里打字，回车发送 |
| 换行（不发送）       | 按住 **Shift** 再按回车 |
| 停止 Aris 正在说的话 | 点击输入框旁边的「**停止**」按钮 |
| 看以前的对话         | 顶部菜单 **视图 → 查看历史记录** |
| 改 Aris 的性格/规则  | 顶部菜单 **视图 → 人设与规则**，在页面里改完点「保存」 |
| 备份/恢复记忆       | 顶部菜单 **文件 → 导出/导入记忆数据库** |

## 若你安装了 Ollama（第一步第 3 项）

- 使用 Aris 前，请**先保持 Ollama 在运行**（安装后通常会自动在后台运行；若没有，可先打开 Ollama 应用，或在终端执行一次 `ollama serve`），再执行 `npm start`。
- `.env` 里若已有 `OLLAMA_HOST=http://localhost:11434` 和 `ARIS_EMBED_MODEL=nomic-embed-text`，无需修改；没有则可自行加上这两行。  
- 若你**跳过了** Ollama，本段可忽略。

---

## 遇到问题怎么办？

- **终端里报错「找不到 npm」或「不是内部或外部命令」**  
  说明 Node.js 没装好或没生效。请重新安装 Node.js，并**重启终端**再试。

- **运行 `npm start` 后窗口没出来或马上闪退**  
  先确认 `.env` 里的 `DEEPSEEK_API_KEY` 已正确粘贴、没有多余空格或引号。再在终端里看是否有红色报错，把报错内容复制下来便于排查。

- **点击中心没有反应**  
  确保点击的是 3D 窗口**内部**的中央区域，而不是标题栏或桌面。

- **Aris 不回复或提示 API 错误**  
  检查密钥是否有效、是否在 DeepSeek 平台有余额或可用额度；确认 `.env` 中 `DEEPSEEK_API_URL` 为 `https://api.deepseek.com`（或你使用的官方地址）。

# 进阶说明（可选阅读）

下面内容适合已经能正常启动、想进一步自定义或开发的用户。

## 需要什么

- **Node.js 18+**
- **DeepSeek API Key**（必选）：在 `.env` 中配置 `DEEPSEEK_API_KEY`
- **Ollama + nomic-embed-text**（可选）：用于本地向量记忆，不配置则记忆能力受限

## 命令速查

| 操作       | 命令 |
|------------|------|
| 安装依赖   | `npm install` |
| 启动应用   | `npm start` |
| 打包成 exe/dmg 等 | `npm run build`（生成在 `dist/`） |

## 可以修改什么（不碰代码）

- **人设与规则**：菜单 **视图 → 人设与规则**，编辑后保存即生效，无需改代码。
- **环境变量**：在 `.env` 中修改 API 地址、Ollama 等。
- **数据与记忆**：通过 **文件 → 导出/导入记忆数据库** 备份或迁移 `.aris` 文件。

## 功能概览

| 功能         | 说明 |
|--------------|------|
| 主界面对话   | 点击中心打开对话层，支持流式输出与停止 |
| 主动发话     | 空闲一段时间后 Aris 可能主动发一句 |
| 历史记录     | 视图 → 查看历史记录 |
| 记忆管理器   | 视图 → 记忆管理器 |
| 人设与规则   | 视图 → 人设与规则 |
| API 提示词预览 | 视图 → API 提示词预览 |
| 导出/导入记忆 | 文件 → 导出/导入记忆数据库 |

## 数据与存储

- 对话与记忆数据存放在应用数据目录（如 `userData/aris/`），可通过 **文件 → 导出/导入** 备份。
- API 密钥仅从 `.env` 读取，不会写进界面或打包进应用。
