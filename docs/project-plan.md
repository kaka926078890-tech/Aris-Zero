# 项目名称：Aris (Alpha Version)

## 1. 项目愿景 (Project Vision)
Aris 是一个运行在桌面端的、冷静且具有独立人格的 AI 交流个体。它不是一个“助手”，而是一个在逻辑层面与用户高度共鸣（MBTI 驱动），且具备**自主记忆与反思能力**的“异体观察者”。
- **视觉核心**：极简 HUD 风格。外层音频环 + 中层方框 + 内核点线粒子系统（Plexus）。
- **人格核心**：基于 INFP 协议，独立、克制、深度，拒绝任何“为您服务”的工具化口吻。
- **智能核心**：感知当前桌面窗口环境（Context），并结合**本地向量记忆**实现具有连续性的同频对谈。

## 2. 视觉规范 (Visual Identity)

### 2.1 空间层级与渲染 (Three.js)
1. **外层 (Outer Ring)**: 
   - 一个巨大的、极细的荧光青 (#00FFCC) 圆环。
   - 它是实时音频示波器，随 AI 语音频率产生向内/向外的尖锐波动。
2. **中层 (Square Frame)**: 
   - 一个静态的、极简的荧光线框正方形，作为核心容器。
3. **内核 (Plexus Core)**: 
   - 位于方框内部，由 15-20 个光点组成的粒子系统。
   - 点与点之间随距离动态连线，形成不断重组、不规则的多面体形态。
   - 动态：缓慢漂移，带有一种“思考中”的律动感。

### 2.2 整体质感
- **风格**：极简工业风、高精细线条（Hairline）、全透明背景。
- **特效**：使用 `UnrealBloomPass` 实现发光效果，但保持线条清晰，严禁模糊嵌套。

## 3. 人格与认知协议 (Personality & Memory)

### 3.1 INFP 独立人格逻辑
- **视角**：平等的对谈者，而非下属。
- **语言**：深邃、简练，能感知情绪并提供逻辑层面的共鸣。
- **触发**：感知到窗口变化或用户主动发起对话。

### 3.2 记忆模块 (M5: Cognitive Memory)
- **长短期记忆结合**：
  - **短期**：当前会话的 Context Window。
  - **长期 (Local Vector DB)**：使用 **LanceDB** 存储用户偏好、重要观点及 **“纠错记录”**。
- **反思能力**：Aris 必须记录用户对其逻辑的否定。如果用户说“你理解错了”，Aris 需在本地记忆中标记该逻辑偏见，并在后续对话中引用、修正。
- **语义检索**：在生成回复前，先检索向量库中“最相关的历史对谈”，确保交流具有生命连续性。

## 4. 技术栈 (Technical Stack)
- **宿主环境**：Electron (透明、置顶、无边框、支持点击穿透)。
- **渲染引擎**：Three.js (Plexus 粒子算法 + 自定义 Shader)。
- **AI 大脑**：DeepSeek API / OpenAI API。
- **存储系统**：
  - **SQLite**: 存储基础设置与原始对话流。
  - **LanceDB**: 嵌入式本地向量数据库，处理语义记忆。
- **音频交互**：Web Audio API (捕获 TTS 音频流频谱)。
- **系统感知**：Node.js 监听活跃窗口标题。

## 5. 核心功能模块 (Core Modules)

### M1: 透明视窗 (Electron Shell)
- 负责窗口穿透属性（setIgnoreMouseEvents）的管理。
- 只有在点击几何体中心区域时，才响应对话输入触发。

### M2: 3D 逻辑核心 (3D Plexus Engine)
- 实现方框内粒子的动态漂移与点距连线算法。
- 外部圆环的 `analyser.getByteFrequencyData` 顶点映射逻辑。

### M3: 认知存储 (Memory & Vector DB)
- 实现对话内容的文本嵌入 (Embedding) 与本地检索。
- 维护“纠错本”，让 Aris 记住并学习用户的逻辑反馈。

### M4: 对谈系统 (Dialogue System)
- 集成 LLM 接口，封装带记忆检索的 Prompt 模板。
- 实现对话气泡 UI (Tailwind CSS)，要求简洁且不遮挡 3D 主体。

## 6. 开发路线图 (Roadmap)

### 第一阶段：视觉框架与透明视窗
- [ ] 搭建 Electron 透明视窗环境。
- [ ] Three.js 绘制外部圆环、中心方框及基础 Plexus 粒子。

### 第二阶段：动态进化与本地存储
- [ ] 实现圆环随音频波动逻辑。
- [ ] **初始化 LanceDB 本地向量库，实现基础的记忆存取功能。**

### 第三阶段：带记忆的 INFP 人格集成
- [ ] 调优包含“历史检索”和“环境感知”的 System Prompt。
- [ ] 实现 AI 的“自我纠错”逻辑：通过对话历史学习并修正回答风格。

### 第四阶段：环境联动与发布
- [ ] 实现窗口标题实时感知。
- [ ] 最终性能调优，确保 3D 渲染与内存占用保持在低位。

## 7. Cursor 开发指令 (Instructions)
1. **独立逻辑**：所有的代码逻辑务必解耦（Engine, Memory, API, Context 分开）。
2. **拒绝工具化**：在编写 Prompt 时，严禁赋予 Aris “助手”身份，它是一个独立观察者。
3. **记忆优先**：任何对话生成前，必须先行检索本地向量库，确保对话具有前后一致的“活人感”。
4. **视觉精度**：线条宽度、发光强度须经由 CSS/Shader 精确控制，追求冷静的电子质感。
